Configuration praticien

Branchez ALIM dans votre IA

Cette page concentre tout ce qui doit rester côté ALIM : clé d'accès, préférences de sortie, règles activées, garde-fous données. Une fois configuré, vous utilisez ALIM depuis votre IA habituelle, sans revenir ici.

Prototype en validation documentaire. Le service de génération de clés et l'API publique seront activés une fois les vérifications HAS / ANSES terminées et la mise en service confirmée. En attendant, demande de clé bêta par e-mail.

01Votre clé ALIM

Une clé API est un secret personnel qui identifie votre installation. Elle s'utilise dans ChatGPT (action OpenAPI), dans Claude (serveur MCP local), ou dans tout client qui appelle l'API ALIM en HTTPS. 

alim_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Préfixe alim_live_ + 32 caractères. La clé est envoyée à chaque appel via Authorization: Bearer ….

  • Création depuis cette page (à venir)
  • Révocation à tout moment
  • Rotation recommandée tous les 6 mois
  • Pas de réutilisation entre praticiens
Demander un accès bêta par e-mail

02ChatGPT À venir

Via un Custom GPT avec action OpenAPI pointant sur l'API ALIM.

  1. Dans ChatGPT, créez un Custom GPT (Explorer → Créer un GPT).
  2. Section Configure → Actions, ajoutez l'action depuis le schéma OpenAPI ALIM (URL fournie quand le service sera publié).
  3. Type d'authentification : API Key, position Bearer.
  4. Collez votre clé alim_live_….
  5. Dans Instructions, ajoutez la consigne ALIM (générée par cette page).
  6. Testez avec un brief de démo.

Consigne système recommandée

Vous êtes un assistant de formulation nutritionnelle pour
diététicien libéral. Pour toute demande de recette ou de menu
clinique, vous DEVEZ appeler l'action generate_clinical_recipe
fournie par ALIM. N'inventez jamais de valeurs nutritionnelles
ni de règles cliniques : présentez uniquement ce qu'ALIM
retourne, en citant les sources et règles activées. Si ALIM
refuse, expliquez le refus traçable plutôt que de contourner.

03Claude (MCP) À venir

Via un serveur MCP local qui appelle l'API ALIM en HTTPS. Même flux d'installation que PennyPilot pour les cabinets comptables : clé saisie une fois, outils disponibles ensuite dans Claude.

Configuration ~/.claude/mcp.json

{
  "mcpServers": {
    "alim": {
      "command": "npx",
      "args": ["-y", "@holco/alim-mcp"],
      "env": {
        "ALIM_API_KEY": "alim_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
        "ALIM_API_URL": "https://alim.care/api"
      }
    }
  }
}

Test après installation

Dans Claude Desktop, vous devez voir l'outil generate_clinical_recipe apparaître dans la liste des MCP servers. Posez par exemple :

Génère un déjeuner végétarien pour une patiente avec
diabète gestationnel à 28 SA, été, équipement plaque
uniquement. Utilise ALIM pour vérifier les glucides, les
sucres ajoutés, les règles grossesse et les valeurs Ciqual.

Claude appellera generate_clinical_recipe, recevra une sortie auditable, et la présentera. Vous gardez la main pour valider avant tout usage clinique.

06Données patient — ce qu'ALIM ne reçoit pas

Aucune donnée patient nominative ne doit transiter par ALIM. ALIM traite uniquement des briefs cliniques anonymes : pathologies, contre-indications, type alimentaire, équipement, saison, budget. Pas de nom, pas de date de naissance, pas d'identifiant patient, pas de numéro de Sécurité sociale, pas de note libre nominative. Si vous écrivez un brief dans ChatGPT ou Claude, retirez toute information identifiante avant l'envoi.

Ce que vous pouvez écrire dans votre IA pour appeler ALIM :

  • « Femme enceinte 28 SA, diabète gestationnel, végétarienne, allergie aux fruits à coque, été »
  • « Homme adulte, diabète T2 + HTA, fonction rénale normale, sans allergie, dîner »

Ce qui ne doit jamais y figurer :

  • Nom, prénom, initiales identifiables, date de naissance précise
  • Numéro de dossier patient, identifiant cabinet, identifiant Sécurité sociale
  • Coordonnées (adresse, téléphone, e-mail)
  • Détails contextuels permettant la ré-identification (« patiente de Mme Dupont à Rennes »)

ALIM ne stocke pas les briefs reçus côté serveur, sauf logs techniques anonymes (timestamp, identifiant de clé, statut de réponse) à des fins de quota et de débogage. Vous restez responsable de la pseudonymisation des informations envoyées à votre IA.

07Vérifier votre installation

Après installation MCP ou ajout de l'action OpenAPI dans ChatGPT, vous pouvez tester rapidement que la clé est active :

Test endpoint santé

curl -H "Authorization: Bearer alim_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx" \
     https://alim.care/api/v1/me

Doit retourner : { "valid": true, "scopes": [...], "rate_limit": {...} }.

Si ça ne marche pas

  • Vérifiez le préfixe alim_live_ dans la valeur de la clé.
  • Pas d'espace avant/après dans la variable d'environnement.
  • Service côté HOLCO en ligne : alim.care/api/health
  • Sinon : alim@holco.co.
